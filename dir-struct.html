Directory Structure

Check libvirt code directory after download. There are following sub-directory in libvirt root directory:

  • build-aux: The autoconf auxiliary directory.
  • docs: Include the documents of libvirt and libvirt relax-ng schema. The former could be seen on libvirt org site. The later is used for libvirt xml validation, such as virt-xml-validate.
  • example: This directory contains libvirt API usage examples.
  • gnulib: The GNU Portability Library. A library that libvirt relies on.
  • include: Libvirt public header files.
  • m4: GNU M4, a Unix macro processor, is a dependency of GNU autoconf.
  • po: GetText related files, they are used for internationalization and localization.
  • src: The libvirt core library codes.
  • tests: Libvirt unit test files.
  • tools: The codes libvirt supplementary programs, such as virsh, virsh shell completion, libvirt-guests service, libvirt wireshark dissector, libvirt-nss, virt-admin, virt-login-shell, virt-host-validate, virt-pki-validate, virt-xml-validate, virt-sanlock-cleanup.

schema

Libvirt use xml to record or manage resources, including domain, domain snapshot, nodedev, secret, storagevol, capability, interface, nwfilter, nwfilterbinding, network.
The relax-ng schema files define the format of libvirt xml
