What is libvirt

Libvirt is a toolkit to manage virtualization platforms.

What libvirt can do

  • Manage virtualization hypervisors: QEMU/KVM, Xen, Virtuozzo, OpenVZ, VMWare ESX, VMware Workstation/Player/Fusion, VirtualBox, LXC, BHyve, IBM PowerVM, Microsoft Hyper-V
  • Manage host resources: storages, networks, host node devices, network filters, host interfaces

Preparation to review the code

Download the code

Use git to download libvirt source code. 

git clone git://libvirt.org/libvirt.git

Learn git

Git is powerful version-control system for tracking changes in source code. The following git sub-commands are useful for browsing the code.

git log

Git log is to show the commits history of the source code.
Run it directly in the source code directory then get the commits from the last to the first:

git log

By format strings, you can set the format of commits log. This link gives the full format or git log. For example, see the commit hash, author, author date, commit subject:

git log --pretty="format:%H %an %aD %s"

And it can get the commits from one to another:

git log COMMIT1..COMMIT2

By using tags, you can find the commits during two versions. For example, the following command will show the commits between version v4.5.0 to v4.6.0 :

git log v4.5.0..v4.6.0

Man page: git log

git tag

Tags are ref's that point to specific points in Git history. Tagging is generally used to capture a point in history that is used for a marked version release (i.e. v1.0.1).
Run as following in source code dir to show all the tags:

git tag

--contains is to show which tags contains a specific commit:

git tag --contains COMMIT

Man page: git tag

git show

Git show can check the detailed changes of commits.
To show the latest code change of a commit:

git show
